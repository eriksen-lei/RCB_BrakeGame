# Agent Collaboration Guide

## Communication Rule

The user may describe requirements quickly, casually, or with missing details. Do not assume ambiguous intent.

If any part of the request is unclear, risky, or could lead to unwanted file changes, ask the user for clarification before executing.

## Execution Rule

- Confirm scope before creating, deleting, or substantially modifying files.
- Do not overbuild beyond the user's latest request.
- When the user asks for a prototype, implement only the requested core mechanic first.
- If the user corrects direction, follow the newest instruction immediately.
